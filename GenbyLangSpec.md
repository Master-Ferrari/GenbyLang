# Genby 1.0 — техническое задание

## Обзор

Genby — это встраиваемый скриптовый язык, поставляемый как TypeScript-библиотека.
Цель — дать конечному пользователю простой способ собирать строки из переменных,
условий и вызовов заранее зарегистрированных функций (типовой юзкейс: сборка
промптов для LLM).

Библиотека предоставляет:
1. Декларативный конструктор языка (регистрация директив, функций, переменных, enum-ов).
2. Синтаксический анализатор и интерпретатор.
3. DOM-поле ввода с подсветкой синтаксиса, автокомплитом и подсветкой ошибок.

Синтаксический анализатор **единый** для всех конфигураций — реализация
пользователя библиотеки меняет только словарь (директивы/функции/переменные/enum-ы),
а не грамматику.

## Синтаксис языка

### Литералы
- Строки: `"..."`, с поддержкой многострочных переносов.
- Интерполяция в строках: `"hello {name}"`.
- Числа: `42`, `3.14`, `-1`.
- Булевы не имеют литералов — результат сравнения.

### Экранирование в строках
Через `\`. Так как строки отправляются в js, экранирование должно работать так же. Но по сути хватит экранирования " и { } .

### Типы данных

Встроенные теги: `STR`, `NUM`, `BUL`, `ENUM`, `ANY`. Экспортируются из библиотеки.
- `ENUM`-значения — отдельный тип; при интерполяции в строку рендерятся как их имя.
- `NUM` при интерполяции рендерится как текст.
- `ANY` — escape-hatch: аргумент/возврат с `ANY` пропускает проверку типа на checker-слое.

Дополнительно пользователь библиотеки регистрирует **кастомные типы** через `addType(name, options?)`.
Имя кастомного типа становится валидным значением для `ArgSpec.type`, `VariableSpec.type` и `FunctionSpec.returns`.
Значения кастомных типов — любые JS-сущности (массивы, объекты, `null`, инстансы классов): язык не интроспектирует их,
а просто прокидывает через присваивания, вызовы и интерполяцию. У самого языка нет литерального синтаксиса для них —
кастомные значения появляются только как возврат зарегистрированных функций.

Опции при регистрации:
- `describe?: string` — попадает в сгенерированный Markdown-референс.
- `stringify?: (value) => string` — вызывается при интерполяции в строку. По умолчанию `String(value)`.

### Операторы
- `+` — конкат для `STR+STR`, сложение для `NUM+NUM`. `STR+NUM` и все варианты с кастомными типами — ошибка.
- `-`, `*`, `/` — только для `NUM`. Деление на ноль — ошибка рантайма.
- `==`, `!=` — для двух значений одного типа (включая два значения одного и того же кастомного типа; сравнение — по ссылке / `===`).
- `<`, `>`, `<=`, `>=` — только для `NUM`.

### Идентификаторы
`[a-zA-Z_][a-zA-Z0-9_]*`. Зарезервированные имена (директивы/функции/переменные/enum-значения)
определяются конфигурацией — пользователь библиотеки решает какие запретить.
Попытка присвоить зарезервированное имя — ошибка.

### Комментарии
`// до конца строки`. Многострочных нет.

### Присваивание
`name = expression`. Переприсваивание допустимо.
Типизация по первому присваиванию (решить при реализации — либо фиксируется тип,
либо переменная остаётся полиморфной).

### Директивы
`@DIRECTIVE(args...)`. Только на верхнем уровне. Исполняются один раз при загрузке,
не участвуют в основном потоке. Регистрируются через конструктор.

### Функции
`FN_NAME(arg1, arg2, ...)`. Все функции — внешние, зарегистрированные через конструктор.
Все вызовы функций **неявно await** — движок исполняется асинхронно.
Void-функции возможны — их результат нельзя присвоить, только вызвать как statement.

### Условные выражения и управляющие формы
В ядре их нет. Пользователь регистрирует нужные формы (`IF`, `CASE`, `FOR`,
`WHILE`, …) через `addFunction`. Все аргументы вычисляются **лениво**:
handler получает объекты `Arg<T>` и сам решает, когда и сколько раз вызвать
`arg.calc()`. Каждый вызов `.calc()` заново исполняет выражение из места
вызова (вместе с его побочными эффектами — это признанная фича: пользователь
видит ровно столько эффектов, сколько раз сам попросил). Memoization
отсутствует; если значение нужно несколько раз — сохраните его в локальную
переменную в handler-е.

### Пользовательские функции (`name(params) = ( ... )`)
Параметры связываются с выражениями из места вызова как «ленивые проекции»:
каждое чтение параметра в теле заново вычисляет соответствующее выражение
в замороженной на момент вызова области видимости вызывающего. Побочные
эффекты при этом дублируются — это поведение согласовано с правилом
`arg.calc()`. Присваивание параметру (`x = ...`) заменяет проекцию
обычным локальным значением для оставшейся части тела.

### Завершение
Последняя строка программы — `RETURN(expression)`. Обязательно. `RETURN` вне конца — ошибка.
Отсутствие `RETURN` — ошибка. Тип возвращаемого значения — любой. Это тоже функция, но вшитая в язык.

По умолчанию `RETURN` принимает значение любого типа. Пользователь библиотеки может
зафиксировать конкретный тип через `genby.setReturnType(type, options?)` — после этого
тип выражения в `RETURN(...)` обязан совпасть с заявленным (правила совместимости те же,
что и для аргументов функций: `ANY` с любой стороны выключает проверку; для `ENUM`
обязательно указать `enumKey`). Значение `VOID` в `setReturnType` запрещено — `RETURN`
всегда обязан продуцировать значение. Повторный вызов `setReturnType` на одном билдере —
ошибка.

## TypeScript API

### Конструктор

```ts
const genby = new Genby()
genby.addDirective({ name, args, required, handler })
genby.addEnum(enumKey, values: string[])
genby.addVariable({ name, type, enumKey? })
genby.addFunction({ name, args, returns, handler })
genby.addType(name, { describe?, stringify? })
genby.setReturnType(type, { enumKey? })   // optional: lock RETURN type
const langMachine = genby.build()
```

`addType` регистрирует кастомный тип. Имя должно быть валидным идентификатором и не совпадать со встроенными
(`STR`, `NUM`, `BUL`, `ENUM`, `ANY`, `VOID`). После регистрации имя можно использовать в `type`-полях
`ArgSpec`/`VariableSpec`/`FunctionSpec.returns`. Финальная валидация, что все упоминаемые типы зарегистрированы,
происходит в `build()` — поэтому порядок вызовов `addType` / `addFunction` / `addVariable` не важен.

### Пресеты

`addPreset(name)` регистрирует ровно одну готовую функцию под её каноническим именем. Каждый пресет
называется так же, как функция, которую он добавляет (и, где это нужно, попутно регистрирует
поддерживающий тип или директиву). Полный список имён экспортируется как `PRESET_NAMES`
(TS-тип `PresetName`):

| Группа | Пресеты (= имена функций) | Примечания |
| --- | --- | --- |
| control | `IF`, `WHEN`, `UNLESS`, `AND`, `OR`, `NOT`, `EQ`, `NEQ`, `COALESCE`, `CHOOSE`, `CASE` | — |
| loops | `FOR`, `TIMES`, `WHILE` | — |
| arrays | `ARR`, `RANGE`, `SIZE`, `AT`, `FIRST`, `LAST`, `SLICE`, `CONCAT`, `REVERSE`, `PUSH`, `CONTAINS`, `INDEX_OF`, `SPLIT`, `JOIN` | любой массивный пресет автоматически регистрирует тип `ARR`, если его ещё нет |
| cast | `STR`, `NUM`, `BUL`, `INT` | — |
| math | `ADD`, `MUL`, `POW`, `SQRT` | — |
| strings | `UPPER`, `LOWER`, `REPEAT`, `REPLACE`, `LEN` | — |
| async | `FETCH_JSON`, `SHA256`, `SHORT` | `FETCH_JSON` также регистрирует директиву `@API_BASE` |

Пресеты не «включаются по умолчанию» — Genby остаётся минимальным ядром. Регистрация конфликтующих имён
(например, `addPreset('STR')` после ручного `addFunction({ name: 'STR', … })`) бросает ошибку обычного
механизма `assertNameFree`. Пресеты можно комбинировать в любом порядке.

```ts
const g = new Genby()
g.addPreset('IF')
g.addPreset('WHILE')
g.addPreset('RANGE')
g.addPreset('STR')
```

Регистрация enum-ов идёт по внутреннему ключу (`enumKey`). Конечный пользователь
языка этот ключ не видит — он видит только имена значений.

Аргумент функции/директивы описывается: `{ name, type, enumKey?, optional?, rest?, describe? }`.
`type` — строка: один из встроенных тэгов (`STR`/`NUM`/`BUL`/`ENUM`/`ANY`)
либо имя пользовательского типа, зарегистрированного через `addType`.
Вариадики задаются `rest: true` на последнем аргументе.

Handler функции всегда получает кортеж объектов `Arg<T>` (сопоставленных по
позициям с массивом `args`): `plain → Arg<T>`, `optional → Arg<T> | undefined`,
`rest → Arg<T>[]` (единственный слот с массивом дескрипторов под все
хвостовые значения). Сигнатура handler-а типобезопасна за счёт маппинга
`HandlerArgs<DS>`; подставьте свой тип в `TypeValueMap` (через declaration
merging), чтобы `await arg.calc()` для пользовательского тэга возвращал
именно ваш TS-тип, а не `unknown`.

### Runtime

```ts
langMachine.check(program: string): CheckResult
langMachine.execute(program: string, inputs: Record<string, Value>): Promise<Value>
langMachine.inputDom(): GenbyInput
```

`check` — возвращает список синтаксических и семантических ошибок (позиция, сообщение).
`execute` — прогоняет программу, возвращает значение из `RETURN` или бросает ошибку.
`inputs` — значения внешних переменных, зарегистрированных через `addVariable`.

### DOM-компонент

```ts
const input = langMachine.inputDom()
input.element: HTMLElement       // вставляется в DOM вручную
input.getValue(): string
input.setValue(text: string): void
input.onChange(cb: (text: string) => void): Unsubscribe
input.destroy(): void
```

Компонент работает с Solid, но к Solid не привязан — это чистый DOM-элемент,
создаваемый императивно. Интеграция в Solid/React/Vue — обязанность пользователя.

Функциональность:
- Подсветка синтаксиса (строки, числа, идентификаторы разных категорий, операторы, комментарии, директивы).
- Автокомплит имён функций, директив, переменных, enum-значений (в контексте аргумента).
- Автоматическая проверка ошибок на каждое изменение + подчёркивание ошибочных
  фрагментов (`inputDom` функционально заменяет `check`).
- Минимальные CSS-стили через CSS-переменные и именованные классы —
  адаптируются пользователем библиотеки. Красное подчёркивание ошибок — отдельный
  класс, не конфликтующий с классами подсветки.

## Экспортируемые типы

```ts
export { Genby, LangMachine, GenbyInput }
export { STR, NUM, BUL, ENUM, ANY, makeEnumValue, isEnumValue }
export { PRESET_NAMES }
export type {
  Value, EnumValue, Type, BuiltinType,
  ArgSpec, FunctionSpec, DirectiveSpec, VariableSpec,
  EnumValueSpec, EnumSpec, TypeDef, TypeOptions,
  CheckResult, GenbyError, ErrorKind,
  Arg, HandlerArgs, TypeValueMap, ValueOfType, PresetName,
}
```

`Type` — строка: встроенные имена (`STR`/`NUM`/`BUL`/`ENUM`/`ANY`) плюс любые имена, зарегистрированные через `addType`.
`Value` — `unknown`: значения кастомных типов могут быть любыми JS-объектами.

## Ошибки

Все ошибки — объекты с полями `{ line, column, length, message, kind }`.
`kind`: `syntax` | `type` | `unknown_identifier` | `runtime` | `reserved_name` | `missing_return`.
Сообщения — на английском, форматирование — за пользователем библиотеки.

## Что оставлено на этап реализации

- Точная грамматика (PEG/ручной парсер — решить).
- Поведение при переприсваивании с изменением типа.
- Приоритет операторов (стандартный математический, уточнить при реализации).
- Поведение `RETURN` в отсутствие явного значения (void-программа разрешена?).
- Лимиты на длину программы, глубину вложенности, время исполнения.
- Формат и механика всплывающего окна автокомплита.
- Синхронизация скролла/курсора между подсвеченным слоем и input-слоем.
- Политика ошибок времени выполнения во внешних функциях (пробрасывать / оборачивать).
- Локаль чисел при интерполяции (точка или запятая).

# Пример кода на Genby 1.0
```
@HOTKEY(CTRL, SHIFT, F2)
@NAME("Translate selection")
@DESCRIPTION("Translates selected text, warns if too long")

// порог длины, при превышении добавляем предупреждение в промпт
warnLimit = 3000
inputLen = LEN(INPUTTEXT)
isLong = inputLen > warnLimit

// условные куски промпта
languageNote = IF(LANG1 == "auto"
  , ""
  , " from " + LANG1)

lengthNote = IF(isLong
  , " The text is long ({inputLen} chars) — be concise but preserve meaning."
  , "")

// основной промпт с интерполяцией и экранированием
prompt = "Translate text below{languageNote} to {LANG2}.{lengthNote}
Preserve all special symbols, line breaks, and the original style.
Do not add commentary. Wrap the result in \"quotes\".

Text (translate and return everything below this line):
{INPUTTEXT}"

// выбор модели в зависимости от длины
model = IF(inputLen < 300, HAIKU_45, SONNET_46)

// void-функция — логируем факт запуска, результат не используем
LOG("translate_run", model)

out = LLM(model, prompt)

RETURN(out)
```
