# Genby 1.0 — техническое задание

## Обзор

Genby — это встраиваемый скриптовый язык, поставляемый как TypeScript-библиотека.
Цель — дать конечному пользователю простой способ собирать строки из переменных,
условий и вызовов заранее зарегистрированных функций (типовой юзкейс: сборка
промптов для LLM).

Библиотека предоставляет:
1. Декларативный конструктор языка (регистрация директив, функций, переменных, enum-ов).
2. Синтаксический анализатор и интерпретатор.
3. DOM-поле ввода с подсветкой синтаксиса, автокомплитом и подсветкой ошибок.

Синтаксический анализатор **единый** для всех конфигураций — реализация
пользователя библиотеки меняет только словарь (директивы/функции/переменные/enum-ы),
а не грамматику.

## Синтаксис языка

### Литералы
- Строки: `"..."`, с поддержкой многострочных переносов.
- Интерполяция в строках: `"hello {name}"`.
- Числа: `42`, `3.14`, `-1`.
- Булевы не имеют литералов — результат сравнения.

### Экранирование в строках
Через `\`. Так как строки отправляются в js, экранирование должно работать так же. Но по сути хватит экранирования " и { } .

### Типы данных
`STR`, `NUM`, `BUL`, `ENUM`. Экспортируются из библиотеки.
`ENUM`-значения — отдельный тип; при интерполяции в строку рендерятся как их имя.
`NUM` при интерполяции рендерится как текст.

### Операторы
- `+` — конкат для `STR+STR`, сложение для `NUM+NUM`. `STR+NUM` — ошибка.
- `-`, `*`, `/` — только для `NUM`. Деление на ноль — ошибка рантайма.
- `==`, `!=` — для двух значений одного типа.
- `<`, `>`, `<=`, `>=` — только для `NUM`.

### Идентификаторы
`[a-zA-Z_][a-zA-Z0-9_]*`. Зарезервированные имена (директивы/функции/переменные/enum-значения)
определяются конфигурацией — пользователь библиотеки решает какие запретить.
Попытка присвоить зарезервированное имя — ошибка.

### Комментарии
`// до конца строки`. Многострочных нет.

### Присваивание
`name = expression`. Переприсваивание допустимо.
Типизация по первому присваиванию (решить при реализации — либо фиксируется тип,
либо переменная остаётся полиморфной).

### Директивы
`@DIRECTIVE(args...)`. Только на верхнем уровне. Исполняются один раз при загрузке,
не участвуют в основном потоке. Регистрируются через конструктор.

### Функции
`FN_NAME(arg1, arg2, ...)`. Все функции — внешние, зарегистрированные через конструктор.
Все вызовы функций **неявно await** — движок исполняется асинхронно.
Void-функции возможны — их результат нельзя присвоить, только вызвать как statement.

### Условные выражения
Через встроенную форму `IF_THEN_ELSE(cond, a, b)` — реализуется как специальная
форма в интерпретаторе (ленивое вычисление веток). Пользователь библиотеки
регистрирует её как обычную функцию, но движок знает о ней по имени (опция).
Так же пользователь сможет зарегестрировать и `FOR(...)` и `SWITCH(...)`.

### Завершение
Последняя строка программы — `RETURN(expression)`. Обязательно. `RETURN` вне конца — ошибка.
Отсутствие `RETURN` — ошибка. Тип возвращаемого значения — любой. Это тоже функция, но вшитая в язык.

## TypeScript API

### Конструктор

```ts
const genby = new Genby()
genby.addDirective({ name, args, required, handler })
genby.addEnum(enumKey, values: string[])
genby.addVariable({ name, type, enumKey? })
genby.addFunction({ name, args, returns, handler })
const langMachine = genby.build()
```

Регистрация enum-ов идёт по внутреннему ключу (`enumKey`). Конечный пользователь
языка этот ключ не видит — он видит только имена значений.

Аргумент функции/директивы описывается: `{ name, type: STR|NUM|BUL|ENUM, enumKey?, optional? }`.
Вариадики — отдельным флагом на последнем аргументе (решить при реализации).

### Runtime

```ts
langMachine.check(program: string): CheckResult
langMachine.execute(program: string, inputs: Record<string, Value>): Promise<Value>
langMachine.inputDom(): GenbyInput
```

`check` — возвращает список синтаксических и семантических ошибок (позиция, сообщение).
`execute` — прогоняет программу, возвращает значение из `RETURN` или бросает ошибку.
`inputs` — значения внешних переменных, зарегистрированных через `addVariable`.

### DOM-компонент

```ts
const input = langMachine.inputDom()
input.element: HTMLElement       // вставляется в DOM вручную
input.getValue(): string
input.setValue(text: string): void
input.onChange(cb: (text: string) => void): Unsubscribe
input.destroy(): void
```

Компонент работает с Solid, но к Solid не привязан — это чистый DOM-элемент,
создаваемый императивно. Интеграция в Solid/React/Vue — обязанность пользователя.

Функциональность:
- Подсветка синтаксиса (строки, числа, идентификаторы разных категорий, операторы, комментарии, директивы).
- Автокомплит имён функций, директив, переменных, enum-значений (в контексте аргумента).
- Автоматическая проверка ошибок на каждое изменение + подчёркивание ошибочных
  фрагментов (`inputDom` функционально заменяет `check`).
- Минимальные CSS-стили через CSS-переменные и именованные классы —
  адаптируются пользователем библиотеки. Красное подчёркивание ошибок — отдельный
  класс, не конфликтующий с классами подсветки.

## Экспортируемые типы

```ts
export { Genby, LangMachine, GenbyInput }
export { STR, NUM, BUL, ENUM }  // Type enum
export type { Value, ArgSpec, FunctionSpec, DirectiveSpec, VariableSpec, CheckResult, GenbyError }
```

## Ошибки

Все ошибки — объекты с полями `{ line, column, length, message, kind }`.
`kind`: `syntax` | `type` | `unknown_identifier` | `runtime` | `reserved_name` | `missing_return`.
Сообщения — на английском, форматирование — за пользователем библиотеки.

- Формат и механика всплывающего окна автокомплита.

- Политика ошибок времени выполнения во внешних функциях (пробрасывать / оборачивать).
- Локаль чисел при интерполяции (точка или запятая).

# Пример кода на Genby 1.0
```
@HOTKEY(CTRL, SHIFT, F2)
@NAME("Translate selection")
@DESCRIPTION("Translates selected text, warns if too long")

// порог длины, при превышении добавляем предупреждение в промпт
warnLimit = 3000
inputLen = LEN(INPUTTEXT)
isLong = inputLen > warnLimit

// условные куски промпта
languageNote = IF_THEN_ELSE(LANG1 == "auto"
  , ""
  , " from " + LANG1)

lengthNote = IF_THEN_ELSE(isLong
  , " The text is long ({inputLen} chars) — be concise but preserve meaning."
  , "")

// основной промпт с интерполяцией и экранированием
prompt = "Translate text below{languageNote} to {LANG2}.{lengthNote}
Preserve all special symbols, line breaks, and the original style.
Do not add commentary. Wrap the result in \"quotes\".

Text (translate and return everything below this line):
{INPUTTEXT}"

// выбор модели в зависимости от длины
model = IF_THEN_ELSE(inputLen < 300, HAIKU_45, SONNET_46)

// void-функция — логируем факт запуска, результат не используем
LOG("translate_run", model)

out = LLM(model, prompt)

RETURN(out)
```
